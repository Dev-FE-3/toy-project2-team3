## <span id="1">✨ 프로젝트 소개</span>

개발자 급여 관련 정보를 효율적으로 수정하고 관리할 수 있는 웹 기반 서비스.
직원들의 급여 내역에 대한 원활한 의사소통을 도모.

<br>

#### 🎯 개발 목적

1. 급여 정정 프로세스 시간 단축.
2. 객관적 데이터 기록을 통한 급여 결정 투명성 확보.
3. 데이터 축적을 통한 일관된 보상 기준 확립
4. 개발자 만족도 및 유지율 증가.
5. 인사팀 업무 효율성 개선.

<br>

#### 📅 기한

> 2025.02.17 - 2025.03.14

![Image](https://github.com/user-attachments/assets/9d080237-7773-4dd6-805f-68a0291ca199)
<br>

#### ✔️ 1주차 : 기획 및 설계

1. 사용자 요구사항 분석 및 기능 정의서 작성.
2. GitHub 레포지토리 초기 설정 및 브랜치 전략 수립.
3. 사용자 경험을 고려한 와이어프레임 제작.

#### ✔️ 2주차 : 기술 환경 구축

1. Firebase 초기 설정 및 데이터 모델링.
2. 일관된 UI/UX를 위한 디자인 시스템 구축.
3. 재사용 가능한 공통 컴포넌트 개발.

#### ✔️ 3주차 : 핵심 기능 개발

1. UI 퍼블리싱 및 반응형 레이아웃 구현.
2. 주요 기능 로직 개발.
3. Firebase 데이터 연동.

#### ✔️ 4주차 : 완성 및 검증

1. 코드 리팩토링 및 파일/폴더 구조 최적화.
2. 크로스 브라우저 테스트 및 성능 최적화.

<br>

### 💡 주요 기능

![Image](https://github.com/user-attachments/assets/9726117e-033d-4d5b-a808-ca1970fbdae9)

#### ✔️ 회원가입 페이지

- 사용자의 이메일, 이름, 비밀번호를 입력하여 회원 가입.
- 중복된 계정이 잇는 경우 모달창을 통해 사용자에게 알림.
- 유효성 검사를 실시하여 통과하지 못하는 경우 입력 테두리가 경고색으로 변경.

![Image](https://github.com/user-attachments/assets/42894c4e-31f8-4591-a08e-29062a09b664)

#### ✔️ 마이페이지

- 사용자의 인사정보와 급여내역이 한 화면에 표시됨.
- 프로필의 수정 아이콘을 통해 이미지를 변경.

![Image](https://github.com/user-attachments/assets/96312389-8301-41bb-80e2-a98764e27c48)

- 화면 하단의 드롭다운 버튼을 통해 급여일 필터링.
- 급여 명세서 확인 버튼 클릭시 모달을 통해 상세 정보 표시.
- 정정 신청하기 버튼을 통해 급여 정정 페이지로 이동.

![Image](https://github.com/user-attachments/assets/4b296333-d475-4744-bb28-055b9daa18d1)

#### ✔️ 급여 정정 페이지

- 화면 좌측에 급여 정정 신청 내역 표시.
- 화면 우측에 급여 정정 신청서 작성 가능.
- 급여 정정 신청시 화면 좌측 신청내역 리스트에 업데이트.

![Image](https://github.com/user-attachments/assets/53f5b9f4-919e-4712-b41a-a7a075c91293)

#### ✔️ 업무 관리 페이지

- 달력에 사용자의 일정 표시 가능.
- 일정 추가 버튼을 통한 모달창으로 업무 추가 가능.
- 일정 별로 색을 나누어 시각적으로 표시.
- 하루 최대 3개까지 일정 중복 가능.
- 전체 삭제 버튼을 통해 해당 월의 데이터만 삭제.

![Image](https://github.com/user-attachments/assets/2bb502b6-d081-45c8-854d-7d893c75688c)

- 각 날짜 cell 을 통해서도 업무 수정 및 삭제 가능.

FSD 폴더 구조 방식을 사용해 확장성과 유지보수성, 높은 재사용성을 통해 생산성 향상 도모.

toy-project1-team1-salary-rectifier
├─ src
│ ├─ app
│ ├─ assets
│ │ ├─ animations
│ │ ├─ fonts
│ │ └─ images
│ ├─ features
│ │ ├─ auth
│ │ ├─ navbar
│ │ ├─ salary-correction
│ │ └─ work-calendar

#### ✔️ 브랜치

정확하고 효율적인 브랜치 관리를 위한 네이밍을 설정.
브랜치를 페이지 단위로 구분하여 각 페이지를 표현.

📋 브랜치명 예시

📚 feature/workCalendar
📚 feature/mypage

## <span id="5">🌵 커밋 컨벤션</span>

✨ Feat : 기능 (새로운 기능)
🔧 Fix : 버그 (버그 수정)
♻️ Refactor : 리팩토링
🎨 Design : CSS 등 사용자 UI 디자인 변경
🔍 Comment : 필요한 주석 추가 및 변경
💄 Style : 스타일 (코드 형식, 세미콜론 추가: 비즈니스 로직에 변경 없음)
📃 Docs : 문서 수정 (문서 추가, 수정, 삭제, README)
🏁 Test : 테스트 (테스트 코드 추가, 수정, 삭제: 비즈니스 로직에 변경 없음)
✏️ Chore : 기타 변경사항 (빌드 스크립트 수정, assets, 패키지 매니저 등)
⚡ Init : 초기 생성
🩹 Rename : 파일 혹은 폴더명을 수정하거나 옮기는 작업만 한 경우
✂️ Remove : 파일을 삭제하는 작업만 수행한 경우
